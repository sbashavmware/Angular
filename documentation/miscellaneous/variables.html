src/environments/environment.ts

environment
environment: object
Type : object
Default value : { production: false, apiUrls: { 'endPoint': '', 'loginUrl' : '/api/login.json', 'getAppTitle' : '/api/getAppTitle.json', 'getCountryList' : '/api/countryList.json', 'getStatesList' : '/api/stateList.json', 'submitUserDet' : '/api/saveUserProfile.json', 'fetchPatientDetails' : '/api/patientDetails.json', 'languagesList' : '/api/languages.json' } }

The file contents for the current environment will overwrite these during build. The build system defaults to the dev environment which uses environment.ts, but if you do ng build --env=prod then environment.prod.ts will be used instead. The list of which env maps to which file can be found in angular-cli.json.
